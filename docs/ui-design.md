# UI 设计

> ⚠️ **历史设计文档** — 2026-07-15 初始规划阶段编写。本文档描述的 UI 布局和交互已在 v0.1.x 中实现，实际界面可能与原文有差异。以当前源码和前端代码为准。
>
> 基于实体模型草案 | 2026-07-15

---

## 1. 信息架构

```
侧边导航：

  仪表板           — 总览统计
  端点             — 我的端点 + 分享给我的
  Provider         — 我的 Provider
  我的用量         — 我自己消耗的 Token
  分享用量         — 我发出去的 Key 被别人用了多少
  用户             — 仅 Admin 可见
```

## 2. 页面设计

### 2.1 仪表板 `/dashboard`

快速概览，展示我的资产和数据：

```
┌─ 统计卡片 ────────────────────────────────────────┐
│ Provider数 │ Endpoint数 │ Key数 │ 今日Token消耗    │
└───────────────────────────────────────────────────┘

┌─ 最近用量趋势（7天）──────────────────────────────┐
│                                                     │
└─────────────────────────────────────────────────────┘

┌─ 我的 Provider ──────┬── 我创建的 Key（Top 5）───┐
│                       │                            │
└───────────────────────┴────────────────────────────┘
```

### 2.2 端点 `/endpoints`

两个 Tab 切换视角：

**Tab 1：我的端点**（我创建的）

```
 ┌──────────────────────────────────────────────────┐
 │ [+ 创建端点]                                      │
 │                                                  │
 │ ┌─ 端点卡片 ──────────────────────────────────┐  │
 │ │ 名称: "Codex入口"                            │  │
 │ │ URL:  /u/alice/codex                        │  │
 │ │ 协议: openai_responses                      │  │
 │ │ Key:  5 个（点击展开 Key 列表）              │  │
 │ │ [编辑] [管理 Key] [删除]                     │  │
 │ └──────────────────────────────────────────────┘  │
 └──────────────────────────────────────────────────┘
```

**Tab 2：分享给我的**（别人分配了 Key 给我，从而我能看到这些 Endpoint）

```
 ┌──────────────────────────────────────────────────┐
 │ ┌─ 端点卡片（只读）───────────────────────────┐  │
 │ │ 名称: "Bob的入口"                            │  │
 │ │ 来自: bob                                    │  │
 │ │ URL:  /u/bob/general                        │  │
 │ │ 协议: openai_chat                           │  │
 │ │                                              │  │
 │ │ 我的 Key:                                    │  │
 │ │   olr_abc123...  [复制]                      │  │
 │ │   olr_def456...  [复制]                      │  │
 │ │                                              │  │
 │ │ 可用模型: gpt-4o, claude-sonnet-5            │  │
 │ └──────────────────────────────────────────────┘  │
 └──────────────────────────────────────────────────┘
```

点击"我的端点"中的某个端点 → 进入该端点详情页，可以管理 Model 可见性、管理 Key。

### 2.3 Provider `/providers`

只有"我的 Provider"（没有分享的概念，Provider 不直接分享）。

```
 ┌──────────────────────────────────────────────────┐
 │ [+ 添加 Provider]                                 │
 │                                                  │
 │ ┌─ Provider 卡片 ─────────────────────────────┐  │
 │ │ 名称: OpenAI                                │  │
 │ │ 协议: openai_chat, openai_responses         │  │
 │ │ 模型: gpt-4o, gpt-4o-mini, o3              │  │
 │ │ Base URL: https://api.openai.com/v1         │  │
 │ │ [编辑] [管理模型] [删除]                     │  │
 │ └──────────────────────────────────────────────┘  │
 └──────────────────────────────────────────────────┘
```

点击 Provider 进入详情，管理其下的 Model 列表。

创建 Endpoint 时，选择协议，勾选"哪些 Provider 的哪些 Model 对此 Endpoint 可见"。

### 2.4 我的用量 `/usage/my`

我自己消耗的 Token（我使用所有 Key — 包括自己的和别人的 — 产生的用量）。

```
┌─ 统计卡片 ────────────────────────────────────────┐
│ Total │ Input │ Output │ Cache │ 请求次数 │ 模型数  │
└───────────────────────────────────────────────────┘

┌─ 趋势图 ──────────────────────────────────────────┐
│ Token / 时间（按天/小时切换）                        │
└───────────────────────────────────────────────────┘

┌─ Token / Key ───┬── Token / Model（环形图）───────┐
│ （柱状图）        │                                  │
│ 每个 Key 用了多少  │ 哪些模型消耗最多                  │
└──────────────────┴──────────────────────────────────┘

┌─ 明细表 ──────────────────────────────────────────┐
│ 时间 │ Key │ Endpoint │ Provider │ Model │ Input │ Output │ Cache │ Total │
└───────────────────────────────────────────────────┘
```

### 2.5 分享用量 `/usage/shared`  ← 核心页面

**站在店铺老板视角**：我把 Key 分发出去，就是在"铺货"。别人调用就是在"消费"。这个页面就是我的生意仪表板。

#### 第一屏：今日概览

打开页面第一眼看到的，回答"今天生意怎么样"：

```
┌──────────────────────────────────────────────────────────────┐
│                      今天，2026-07-15                         │
│                                                              │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────────┐ │
│  │ 今日消耗  │  │ 较昨日    │  │ 活跃Key   │  │ 活跃客户     │ │
│  │ 1.2M     │  │ ↑ 23%    │  │ 8 / 12   │  │ 5 人        │ │
│  │   tokens │  │  +2.3k   │  │  使用中   │  │  今日有调用  │ │
│  └──────────┘  └──────────┘  └──────────┘  └──────────────┘ │
└──────────────────────────────────────────────────────────────┘
```

- **较昨日**：用颜色区分涨（绿）跌（红）
- **活跃 Key**：今天至少被调用过一次的 Key / 我发的全部 Key

#### 第二屏：趋势

```

```

│                            │                                 │
│  1. bob        420k ████   │  1. gpt-5.5      680k ██████   │
│  2. charlie     380k ███    │  2. claude-sonnet 320k ███     │
│  3. dave        210k ██     │  3. gpt-4o        150k █       │
│  4. ...                    │  4. ...                        │

每个排行项显示：排名、名称、消耗量、迷你进度条。

#### 第四屏：Key 健康度

我发出的所有 Key 的表格，一眼看出哪个 Key 有问题：

```
┌─ Key 状态 ──────────────────────────────────────────────────┐
│                                                              │
│  Key名称      分配给   最近调用     今日消耗   状态           │
│  ───────────  ───────  ──────────  ────────  ──────         │
│  Bob日常       bob     10分钟前    420k      🟢 活跃         │
│  Charlie工作   charlie 2小时前     380k      🟢 活跃         │
│  Dave测试      dave    5小时前      85k      🟡 低频         │
│  Eve临时       eve     3天前         0       🔴 沉默         │
│  Frank备用     frank   从未使用       0       ⚪ 未激活       │
│                                                              │
│  [筛选: 全部 | 活跃 | 低频 | 沉默 | 未激活]                    │
└──────────────────────────────────────────────────────────────┘
```

状态定义：
- 🟢 活跃：今天有调用
- 🟡 低频：3 天内有调用，但今天没有
- 🔴 沉默：3-7 天无调用
- ⚪ 未激活：从未被使用过

点击某个 Key → 展开该 Key 的详细消费记录。

## 3. 导航结构

```
侧边栏：

  📊 仪表板       /dashboard
  🌐 端点         /endpoints
  🖥 Provider     /providers
  📈 我的用量     /usage/my
  📤 分享用量     /usage/shared
  👤 用户         /users          ← 仅 Admin

底部：用户头像、用户名、角色标签、退出按钮
```

---

## 4. 页面间导航

```
仪表板
  ├── 点击 Provider → /providers（展开具体 Provider）
  ├── 点击 Key → /endpoints（展开具体 Endpoint 的 Key 列表）

端点
  ├── 点击"我的端点"卡片 → /endpoints/:id（管理 Model 可见性 + Key）
  │     └── 点击某个 Key → 显示 Key 详情（复制、分配对象、用量）
  ├── 点击"分享给我的"卡片 → 只读视图

Provider
  └── 点击 Provider → /providers/:id（管理 Model 列表）

我的用量
  └── 趋势图 + Key/Model 分布 + 明细

分享用量
  └── 每个 Key 被用了多少 + 谁在用 + 明细
```
